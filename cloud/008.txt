FusionCompute安装与配置：
    安装：
        安装准备：
            安装PC或便携: 
                内存: >= 2GB (可用内存1G )
                硬盘：操作系统所在磁盘分区剩余空间大于1GB，至少有一个非操作系统所在的磁盘分区剩余空间大于5GB；( 安装I具所在的磁盘分区)
                操作系统：Windows XP/Windows 7/Windows Server 2003/Windows Server 2008；
                分辨率:推荐1280x1024及以上；


            服务器CNA：
                CPU：支持硬件虚拟化技术,如InteI的VT-x或AMD的AMD-V，并开启；
                内存：最低≥8GB，推荐内存配置≥48GB；
                硬盘：≥16GB；
                Raid：建议使用1、2号硬盘组RAID 1，提高主机可靠性；

            注意事项：
                安装过程不允许修改本地PC或便携的ip地址；
                安装时需要暂时关闭本地PC的防火墙与杀毒软件；
                使用安装工具前请先使用FusionCompute_EnvCheck.bat检查本地PC系统是否满足运行条件；
                搭建环境过程中，不允许更换本地PC机；
                共享文件夹路径中不能包含中文字符，完整路径长度不超过256位；
                环境搭建过程，不要随意对主机进行重启操作；

        安装流程：
            开始 --> 安装主机 --> 安装VRM --> 结束；
        
                主机安装：
                    建议使用1、2号硬盘组RAID 1；
                    通过PXE方式批量安装主机；
                    通过挂载ISO安装镜像方式手动安装主机；
            
                VRM安装:
                    在虚拟机上部署VRM，可使用FusionCompute安装向导完成部署；
                    在物理服务器上安装VRM,需要手动挂载VRM的ISO安装镜像文件并手动完成安装和配置，过程类似通过挂载ISO镜像方式安装主机；

            安装CNA：
                PXE方式安装：通过PXE方式批量安装主机，需要在本地PC部署DHCP服务和FTP服务，通过PXE方式为所有待安装主机自动安装操作系统；

                ISO方式安装：在network中的gateway是必选配置，如果未配置在使用工具安装VRM时会失败（可安装好后使用hostconfig修改）；

            VRM安装：
                部署原则：
                    VRM虚拟化部署场景下，VRM节点部署在由管理集群的指定主机创建的虚拟机上，物理部署场景下,VRM节点部署在物理服务器上；
                    主备部署时需要将主备VRM节点分别部署在两台管理集群主机上；

                虚拟化：
                    使用虚拟机部署的VRM，可以使用FusionCompute安装向导，在指定的主机上通过VRM模板部署VRM虛拟机；
                    一般的，仲裁IP选择管理平面的网关，或AD、DNS等与管理平面互通的全局服务器IP；

                VRM物理服务器部署流程:
                    开始 -> 安装VRM节点 -> VRM是否主备部署? -> 安装第二台VRM节点 -> 配置VRM节点主备关系 -> 结束；

    配置：
        配置流程：
            工程期间可申请调测License没有License还可以使用90天；
            配置主机时注意绑定网口,主备或负荷分担,同时需要注意交换机配；
            FusionSphere支持IPSAN , FCSAN , NAS ,FusionStorage和本地磁盘等多种存储设备类型,具体的配置操作请注意查看资料；
            IPSAN时需要配置主机存储接口, -般不建议网卡绑定,推荐存储多路径；
            在FusionSphere解决方案中,无需在业务平面的分布式交换机下创建端口组, FM会做；
            NTP很重要,如果用域名,那么还需要DNS；

        集群配置：
            创建集群：
                基本属性：集群基本信息设置和集群资源调度策略设置；
                高级属性：集群的HA策略、内存复用策略、GuestNUMA策略、异构CPU迁移策略；

            添加主机：
                配置参数:主机名称，IP地址，描述，BMCIP， BMC用户名，BMC密码
                添加主机的方式:手动添加、使用模板批量添加。
                手动添加:操作简单，但每次只能添加一台主机，适用于主机数量较少的场景。
                使用模板批量添加:可在模板中设置所有主机信息，并一次性导入，适用于主机数量较多的场景；

            网口绑定：为了提高网络的可靠性，推荐管理平面、业务平面的网口分别进行绑定；

            主机添加存储接口：
                主机添加存储接口：实现主机与存储设备对接;添加多个存储接口，可以实现存储的多路径传输；
                加存储接口的方式：手动添加存储接口和批量添加主机存储接口；
                  参数:名称、描述、IP地址、子网掩码、VLAN ID、限速设置、上限 带宽；

        存储配置：
            添加存储资源：为系统添加存储资源，如果系统中主机均使用本地存储或FC SAN存储，无需执行此任务；
            
            配置存储路经：
                系统默认使用通用多路径软件，非华为存储设备，无需更改；
                使用华为的iSCSI、光纤通道存储设备，选择为华为多路径，以提升存储多路径性能；

            主机关联存储资源：
                批量关联存储资源：适用于主机数量较多，或每台主机需要关联多个存储资源的场景；
                手动关联存储资源：适用于主机数量较少，且每台主机仅需要关联一-个或少数存储资源的场景；

            扫描存储设备：发现可以供主机使用存储设备；
            添加数据存储：
                数据存储类型：虛拟化、非虚拟化和裸设备映射；
                    
                    一个主机可以添加多个数据存储，一个数据存储也可以添加到多个主机上，只有当虚拟机磁盘所在的数据存储同时添加
                    到两个主机上时，虚拟机才能在这两个主机之间进行计算迁移；
                    
                    已选择虚拟化的存储，不可逆为非虛拟化存储；


        网络配置：
            创建分布式交换机：
                分布式交换机的功能类似于普通的物理交换机，每台主机都连接到分布式交换机中，分布式交换机的一端是与虚拟机相连的
                虚拟端口，另-端是与虚拟机所在主机上的物理以太网适配器相连的上行链路，通过它可以连接主机和虚拟机，实现系统网络互通；

            配置VLAN池：
                按照规划添加VLAN池，为创建端口组提供VLAN资源；
                当端口组选择连接方式为VLAN池，需要用户手动配置VM的IP地址；

            配置子网：添加子网，当端口组选择连接方式为子网时，系统会根据子网中配置的IP地址池，为使用该端口组的虚拟机自动分配IP地址；

            添加上行链路：
                添加上行链路:在已创建的分布式交换机中添加.上行链路，为虚拟机提供网络资源；
                上行链路为服务器的物理网口，可在添加之前对其进行端口绑定(主备或负荷分担)；
                上行链路的添加需要根据所在节点上VM的需要灵活配置，一般的，上行链路可分为管理，存储和业务三个平面；

            创建端口组：
                端口组是-种策略设置机制，这些策略用于管理与端口组相连的网络。虛拟机的虚拟网卡连接到分布式交换机的端口组，
                与同一端口组相连接的虚拟机各自在不同的主机上，这些虚拟机也都属于虛拟环境内的同一网络；
                
                注意:如果选择了DHCP隔离，或IP与MAC绑定，则不能在VM首次获取IP后进行修改，否则将无法通信；


        系统配置：
            配置NTP：
                设置主机时间同步需要重启主机业务进程，会引起主机服务暂时中断，需谨慎操作；
                
                建议优先配置外部NTP时钟源。如果不存在外部NTP时钟源，则建议将时钟源设置为VRM节点所在主机(虛拟化部署时)或VRM节点(物理部署时)；
                
                如果不配置系统的时间同步，则需手动设置每台主机的时间，保证所有主机间时间与VRM节点的时间误差在1分钟以内，
                否则FusionCompute将无法对时间误差大于1分钟的主机进行监控；
                
                配置的系统时区为导出告警信息显示时间所属的时区，在FusionCompute中，显示的时间为浏览器所设置时区的时间；

            配置虚拟机使用的Mac地址段：
                该任务指导软件调测工程师在完成FusionCompute的安装后，为系统配置自定义的MAC地址段，用于为虚拟机分配唯一的MAC地址；
                
                如果直接使用系统提供的默认地址段(28:6E:D4:88:C6:29~ 28:6E:D4:8A:39:40)，则无需执行本任务；
                
                FusionCompute提供了1 0万个MAC地址供用户使用，地址段为28:6E:D4:88:B2:A1~ 28:6E:D4:8A:39:40,
                其中前5000个地址(28:6E:D4:88:B2:A1~ 28:6E:D4:88:C6:28)专供VRM虛拟机使用；
                
                配置自定义的MAC地址段时，需要将默认地址段修改为自定义地址段，或添加新的地址段修改MAC地址AC地址不能重复；
